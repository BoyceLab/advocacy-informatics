## Welcome

This training site introduces biomedical and public health informatics students to the growing role of **patient advocacy groups (PAGs)**, **patient-led research**, and **engagement-centered health informatics**. The curriculum is organized using Harold Lehmann's **Informatics Stack** as its structural framework — a nine-level heuristic that moves from foundational technology up through organizational purpose and societal context.

By the end of this course, you should be able to:

- Apply the Informatics Stack to patient-facing health information systems
- Understand how PAGs use informatics tools to build registries and drive research
- Explain how regulatory frameworks (21st Century Cures Act, PCORI, REMS) shape patient data access
- Identify real-world success stories from the ALS and rare epilepsy communities
- Evaluate collaborative research readiness using the CMSS/PLRC Scorecard framework

## How to Use This Site

Each module is self-contained but designed to build on the previous. We recommend moving through the **Informatics Stack** module first — it provides the conceptual vocabulary used throughout.

!!! tip "Instructor Note"
    Each module includes a **Stack Mapping** sidebar that shows which levels of Lehmann's framework are most relevant. Encourage students to consider how a PAG's work spans *all* levels — from hardware (wearables) up through world-level policy (Cures Act).

## Key Frameworks Referenced

| Framework | Source | Role in This Curriculum |
|---|---|---|
| **Informatics Stack** | Harold Lehmann, Johns Hopkins | Structural framework for all modules |
| **CMSS/PLRC Scorecards** | CMSS + Patient-Led Research Collaborative (PCORI-funded) | Evaluating collaboration quality |
| **PCORI Engagement Standards** | PCORI | Guiding principles for patient engagement |
| **21st Century Cures Act** | U.S. Congress (2016) | Regulatory baseline for data access |
| **HL7 FHIR / CodeX** | HL7 International | Technical interoperability standard |
